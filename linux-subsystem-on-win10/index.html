Windows 10 has a Linux Subsystem

Last year, Microsoft has introduced “Windows Subsystem for Linux” (WSL) onto Windows 10 - 64 bit edition. It was primarily targeted at open source developers who needed to develop on Windows, but were required to use command line tools which were developed for *nix first. Additionally, owing to the large differences between Windows and *nix filesystems, creating a Windows port for these projects was a low priority given the rapid development. A good example of this is Tensorflow where releases pre-1.0.0 did not have a windows release. Since the API has stabilised, and there is money to be made across platforms, Google has since created a Windows install path.

The release of .NET Core has opened up a world of new possibilities for developers previously limited to the Windows ecosystem. Applications developed with .NET Core can also be deployed on popular Linux distributions. The catch is that former Windows only developers need now to up their *nix game…

WSL is a good starting point. Here’s how to get started…

  1. Open the Windows Settings application, and select the Update and Security option.

  2. Select For Developers and switch Developer Mode on.

  3. Open the windows Control Panel and select Programs and Features. Add the Linux Subsystem feature as shown.

    After selecting OK, Windows will need to reboot.

  4. Following the reboot, select the “Bash” option from the start menu. It will be faster to type-to-search for the command then to dig it out of the menus.

    A command prompt will open with a URL:

    Opening the URL will in-turn open a windows store page offering a choice of Linux distributions to install.

    For this demonstration we’ll install Ubuntu.

    After selecting the GET button Ubuntu will begin the download and install - approx 195Mb.

    You’ll be prompted for a new username and password for the Linux Subsystem

    And we’re ready to go…

  5. Your familiar windows drives can be found from the root directory as: /mnt/c etc.

In a future article we’ll see what we can do with this beyond standard linux file management.
